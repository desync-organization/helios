# Emergency, Offline Fallback, and Demo Recovery

On pause, lease loss, base conflict, provider outage, secret detection, or ambiguous write-back, stop
new work and preserve the canonical cursor. Never retry a mutation from the gateway. Resume only after
Member 2 confirms policy/lease state; replay safe redacted events by cursor and label them `replayed`.

For judging, stop training, warm the selected verified models, confirm rollback, run one task per mode,
and open only reachable frontend surfaces plus real result URLs. Keep a labelled fallback replay for
network failure, but never count it as live completion. Record two timed rehearsals in
`evidence/rehearsal-log.md`.
